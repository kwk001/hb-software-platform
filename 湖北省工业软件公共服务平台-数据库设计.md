## 一、设计规范

### 1.1 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 数据库 | 小写，下划线分隔 | `hubei_industrial_platform` |
| 表名 | 小写，下划线分隔，复数形式 | `users`, `enterprises` |
| 字段名 | 小写，下划线分隔 | `user_id`, `created_at` |
| 索引名 | `idx_表名_字段名` | `idx_users_mobile` |
| 外键名 | `fk_表名_关联表名` | `fk_enterprises_user_id` |

### 1.2 通用字段规范

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| id | BIGINT UNSIGNED | 主键，自增 | AUTO_INCREMENT |
| created_at | DATETIME | 创建时间 | CURRENT_TIMESTAMP |
| updated_at | DATETIME | 更新时间 | CURRENT_TIMESTAMP ON UPDATE |
| is_deleted | TINYINT | 软删除标记 0=否 1=是 | 0 |
| create_by | BIGINT UNSIGNED | 创建者ID | NULL |
| update_by | BIGINT UNSIGNED | 更新者ID | NULL |

### 1.3 状态字段规范

| 模块 | 字段名 | 说明 |
|------|--------|------|
| 用户/企业 | status | 0=已注册,1=待审核,2=审核中,3=已入驻/已通过,4=已驳回 |
| 软件 | status | 0=草稿,1=待审核,2=已上架,3=已驳回,4=已下架,5=更新待审 |
| 需求 | status | 0=待处理,1=处理中,2=已处理,3=已关闭 |
| 补贴券 | status | 0=待审核,1=审核通过,2=已驳回,3=已使用,4=已退还 |
| 留言 | status | 0=待处理,1=处理中,2=已回复,3=已关闭 |
| 政策 | status | 0=草稿,1=已发布,2=已下架 |

## 三、核心模块表结构

### 3.1 企业入驻与审核模块

#### 3.1.1 用户表（users）

```sql
CREATE TABLE `users` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '用户ID',
  `mobile` VARCHAR(11) NOT NULL COMMENT '手机号',
  `password` VARCHAR(255) NOT NULL COMMENT '加密密码（BCrypt）',
  `enterprise_type` TINYINT NOT NULL DEFAULT 0 COMMENT '企业类型：1=工业软件企业,2=工业制造企业',
  `status` TINYINT NOT NULL DEFAULT 0 COMMENT '状态：0=已注册,1=待审核,2=审核中,3=已入驻,4=已驳回',
  `last_login_at` DATETIME DEFAULT NULL COMMENT '最后登录时间',
  `last_login_ip` VARCHAR(50) DEFAULT NULL COMMENT '最后登录IP',
  `login_fail_count` INT DEFAULT 0 COMMENT '登录失败次数',
  `locked_until` DATETIME DEFAULT NULL COMMENT '账号锁定截止时间',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `is_deleted` TINYINT NOT NULL DEFAULT 0 COMMENT '是否删除：0=否,1=是',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_users_mobile` (`mobile`),
  KEY `idx_users_status` (`status`),
  KEY `idx_users_enterprise_type` (`enterprise_type`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';
```

```sql
  `city` VARCHAR(50) NOT NULL COMMENT '城市',

  `is_deleted` TINYINT NOT NULL DEFAULT 0 COMMENT '是否删除：0=否,1=是',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_enterprises_user_id` (`user_id`),
  UNIQUE KEY `uk_enterprises_credit_code` (`credit_code`),
  KEY `idx_enterprises_name` (`enterprise_name`),
  KEY `idx_enterprises_industry` (`industry`),
  CONSTRAINT `fk_enterprises_user_id` FOREIGN KEY (`user_id`) REFERENCES `users` (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='企业信息表';
```

#### 3.1.3 入驻审核记录表（enterprise_audits）

```sql
CREATE TABLE `enterprise_audits` (

```
