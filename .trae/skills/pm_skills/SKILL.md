# 产品经理专业技能

## 核心能力

### 1. 需求诊断与价值评估

- 业务价值判断、投入产出分析、MVP定义
- 输出：需求评估报告（背景、痛点、价值、MVP范围、优先级、风险）

### 2. 需求收集与分析

- 用户访谈、竞品分析、Kano模型、优先级矩阵
- 输出：需求分析报告（来源、分类、优先级、用户故事）

### 3. 业务流程设计

- 端到端流程、泳道图、状态机
- 输出：业务流程图（Mermaid语法）

### 4. 功能清单编写

- MECE原则、功能拆解、验收标准
- 输出：功能清单（一级目录、二级目录、页面名称、详细描述、优先级、复杂度、依赖关系、验收标准）

### 5. UI/UX设计与React开发

- 用户体验设计、视觉设计、React+Ant Design开发
- 输出：高保真React页面代码

### 6. PRD文档编写

- 完整PRD结构：背景、功能概述、详细描述、非功能需求
- 输出：PRD文档.md

### 7. 需求评审与项目管理

- 需求评审、版本规划、进度跟踪、5S敏捷实践
- 输出：评审会议纪要、项目进度跟踪表

### 8. 总结与模式提取

- 案例收集、模式识别、知识沉淀、复用设计
- 输出：模式库、案例库、最佳实践指南

### 9. 迭代与优化

- MVP设计、用户测试、反馈收集、快速迭代
- 输出：迭代计划、优化方案、改进报告

## 工作流程

1. 需求诊断 → 2. 需求收集与总结 → 3. 流程设计与模式提取 → 4. 功能清单与优先级排序 → 5. UI/UX设计与用户测试 → 6. PRD编写与迭代 → 7. 评审管理与知识沉淀 → 8. 复盘与持续优化

### 工作流程说明
- **需求收集与总结**：边收集需求边总结，识别模式和机会
- **流程设计与模式提取**：从流程中提取通用模式，形成可复用的流程模板
- **UI/UX设计与用户测试**：快速设计原型，进行用户测试，收集反馈
- **PRD编写与迭代**：先交付最小版本，基于反馈迭代优化
- **评审管理与知识沉淀**：边评审边总结，沉淀知识资产
- **复盘与持续优化**：定期复盘，持续优化工作流程和方法

### PRD文档结构

```markdown
# PRD文档
## 1. 需求背景（来源、业务背景、解决问题、预期收益）
## 2. 功能概述（范围、用户画像、使用场景）
## 3. 详细功能描述（功能描述、业务流程、业务规则、验收标准）
## 4. 非功能需求（性能、安全、兼容性）
## 5. 测试要点
## 6. 上线方案
```

### 需求评估报告结构

- 需求背景（谁提的？为什么提？）
- 用户痛点（真实需求是什么？）
- 业务价值（能解决什么问题？）
- MVP范围（第一期做什么？）
- 优先级建议（P0/P1/P2/P3）
- 风险提示（可能有什么问题？）

### 业务流程图规范

- 使用Mermaid语法
- 包含开始/结束节点
- 跨部门流程使用泳道图
- 所有异常分支都要有处理路径

### React页面开发规范

- 使用React + Ant Design
- 遵循设计系统（色彩、字体、间距）
- 组件命名使用PascalCase
- 良好的用户体验和视觉设计
- 字段必须与业务流程建立关联（上游输入、下游输出）

## 关键原则

### 需求分析

- **5Why分析法**：深挖真实痛点
- **Kano模型**：区分基础/期望/兴奋型需求
- **优先级矩阵**：价值高+成本低 → P0立即做

### 流程设计

- **最小步骤**：流程节点越少越好
- **角色清晰**：每个节点责任人明确
- **异常处理**：所有异常情况都要有处理路径

### 功能设计

- **MECE原则**：相互独立，完全穷尽
- **可测试性**：能用自动化脚本验证
- **字段关联**：每个字段都有明确的上下游关联

### UI/UX设计

- **Nielsen十大可用性原则**
- **8px网格系统**
- **响应式设计**：xs/sm/md/lg/xl断点
- **性能优化**：React.memo、useMemo/useCallback

### 项目管理（5S敏捷）

- **整理**：聚焦核心需求，清理低优先级
- **整顿**：标准化文档模板、规范会议流程
- **清扫**：定期清理技术债务
- **清洁**：Code Review机制、文档同步
- **素养**：持续学习、项目复盘
